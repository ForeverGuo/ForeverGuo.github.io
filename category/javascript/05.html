# 单例模式

  1. 应用
  • 登录框
  • 购物车
  • vuex 和 redux的store
  1. 设计原则验证

  • 没法具体开放封闭原则，但是绝对不违背开放封闭原则
  1. 代码

# 适配器模式

  1. 应用
  • 封装旧接口
  • vue的 computed
  1. 设计原则验证
  • 将旧接口和使用者进行分离

  1. 代码
class Adaptee {
  specificRequest() {
    return `德国制造`;
  }
}

class Target {
  constructor() {
    this.adaptee = new Adaptee();
  }

  request() {
    let info = this.adaptee.specificRequest();
    return `${info} - 转换器 - 中国制造`
  }
}

const target = new Target();
target.request();

function log(target, name, describe) {

    console.log(`calling ${name} with`, arguments);
    return oldValue.apply(this, arguments);

}

# 代理模式

  1. 应用
  • 网页事件代理
  • jquery $.proxy
  • ES6 Proxy
  1. 设计规则验证
  • 代理类和目标类分开，隔离开目标类和使用者
  • 符合开放封闭原则
  1. 代码演示
class ReadImg {
  constructor(filename) {
    this.filename = filename;
    this.loadImage();
  }
  loadImage() {
    console.log('loading... ' + this.filename);
  }
  display() {
    console.log('display... ' + this.filename);
  }
}

class ProxyImage {
  constructor(filename) {
    this.realImg = new ReadImg(filename);
  }
  display() {
    this.realImg.display();
  }
}

let initImage = new ReadImg('1.png');
initImage.display()

# 对比

  • 代理模式 提供一模一样的接口 显示原有功能，但是是经过限制和处理的

  • 适配器模式 提供一个不同的接口

  • 装饰器模式 扩展功能，原有功能不变且可直接使用

# 外观模式

  1. 应用
  • 根据参数处理来实现返回特定需求的接口
  1. 设计原则验证
  • 不符合单一职责原则和开放封闭原则，因此谨慎使用，不可滥用

# 观察者模式

  1. 应用
  • 网页事件绑定
  • Promise
  • jQuery callbacks
  • nodejs 自定义事件
  • nodejs 处理http请求，多进程通信
  • vue 和 react 组件生命周期触发
  • vue watch
  1. 设计原则验证
  • 主题和观察者分离，不是主动触发而是被动监听，两者解耦
  • 符合开放封闭原则
  1. 代码演示
//  主题
class Subject {
  constructor() {
    this.state = 0;
    this.observers = [];
  }
  getState() {
    return this.state;
  }
  setState(state) {
    this.state = state;
    this.notifyAllObservers();
  }
  notifyAllObservers() {
    this.observers.forEach(observe => {
      observe.update()
    })
  }
  attach(observe) {
    this.observers.push(observe);
  }
}

class Observe {
  constructor(name, subject) {
    this.name = name;
    this.subject = subject;
    this.subject.attach(this);
  }
  update() {
    console.log(`name: ${this.name}, state: ${this.subject.getState()}`)
  }
}

let s = new Subject();
let o1 = new Observe('o1', s);
s.setState(1);

# 迭代器模式

  1. 应用
  • jQuery each
  • es6 Iterator
  1. 设计原则验证
  • 迭代器对象和目标对象分离
  • 迭代器将使用者和目标对象隔离开
  • 符合开放封闭原则
  1. 代码演示
class Iterator {
  constructor(container) {
    this.list = container.list;
    this.index = 0;
  }
  next() {
    if(this.hasNext()) {
      return this.list[this.index++];
    }
    return null;
  }
  hasNext() {
    if(this.index >=  this.list.length) {
      return false;
    }
    return true;
  }
}

class Container {
  constructor(list) {
    this.list = list;
  }
  // 生成遍历器
  getIterator() {
    return new Iterator(this);
  }
}

let container = new Container([1,2,3,4,5,6])
let iterator = container.getIterator();
while(iterator.hasNext()) {
  console.log(iterator.next())
}

  1. es6 的 Iterator 为何存在 ？
  • es6 语法中，有序集和的数据类型有很多
  • Array Map Set String TypedArray Arguments NodeList
  • 需要有一个统一的遍历接口来遍历所有数据类型
  • 注意： Object 不是有序集合，可以用 Map 来代替
  1. Iterator 是什么
  • 以上数据类型，都有[Symbol.iterator] 属性
  • 属性值是函数，执行函数返回一个迭代器
  • 这个迭代器有 next 方法，可顺序迭代子元素
  • 可运行Array.prototype[Symbol.iterator] 来测试
  • generator 函数返回的就是 iterator 数据接口

# 状态模式

  1. 应用
  • 有限状态机 （Promise）
  • javascritp-state-mechine（开源库）
  1. 设计原则验证
  • 将状态对象和主题对象进行分离，状态的逻辑变化单独处理
  • 符合开放封闭原则
  1. 代码演示
class State {
  constructor(color) {
    this.color = color;
  }
  handle(context) {
    console.log(`turn light to ${this.color}`)
    context.setState(this)
  }
}

class Context {
  constructor(state) {
    this.state = null
  }
  getState() {
    return this.state;
  }
  setState(state) {
    this.state = state;
  }
}

let context = new Context();

let red = new State('red');

red.handle(context);
console.log(context.state)

# 原型模式

  1. 应用
  • js 中的Object.create();
  1. 设计原则验证
  • 不同对象的输出
  1. 代码演示
const prototype = {
  getName: function() {
    return this.first + ' ' + this.last;
  },
  say: function() {
    alert('hello')
  }
}

// create 实现方式
function create() {
  const obj = {};
  let constructor = [].shift.call(arguments);
  if(typeof constructor == 'object') {
    return constructor;
  }
  obj.__proto__ = constructor.prototype;
  let result = constructor.apply(obj, arguments);
  return typeof result == 'object' ? result : obj;
}

const x = Object.create(prototype);
x.first = 'A';
x.last = 'B';
console.log(x.getName())

# 组合模式

  1. 应用
  • 虚拟 DOM 中的 vNode 的数据类型，进行单一类型组合
  • 业务类型 实现菜单

  • 符合开放封闭原则
