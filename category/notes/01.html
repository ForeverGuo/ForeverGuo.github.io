自我管理 -- 01

grantguo 2022-05-08 Manager

# 作为一名CODER的疑问 ？？

  1. 管理是什么，管理者又是什么～
  2. 管理的正确方式是什么～
  3. 凭什么管理别人，别人又凭什么让你管理～
  4. 和管理自己的代码一样么～

带着以上的问题，进入了第一节课的学习～

# 管理素养

  1. 以身作则胜千言
  2. 通过别人拿结果，通过结果培养人
  3. 心要仁慈，刀要快
  4. 选拔比培养更重要
  5. 大胆使用是最好的培养
  6. 教是最好的学（创造学习氛围）
  7. 苛求过程，释怀结果
  8. 没有过程的结果是垃圾，没有结果的过程是放屁
  9. 丑话当先，no surprise
  10. 跟我上，还是给我上 ？

如果你对以上10条中有非要认同的，那么你已经入门了，这是一个管理者的基本素养，我非常认同 1，7，9，而我一直也是秉持着以身作则胜千言这句话，无论是对自己还是对其他人，当然适当的沟通还是有必要的，做一件事先不要想结果的好坏，结果不好怎么办，最起码要付诸于行动，过程中你会获得无限收获，对于第9条说的是对于team的小伙伴的，我们做最好的自己，任何困难也就不是困难了，切记：不要始于言，止于行。

# 领导者即管理者

  • 使命

    1. 企业而言：愿景 => 价值观 => 商业设计 => 运营目标 => 目标到执行体系 => 业务营运保障体系

    2. 个人而言：定目标 => 追过程 => 拿结果

    这里所说的针对于个人而言的总结，其实是针对管理的总结，但是我个人理解，还是先归结为个人比较好，目标也分为不同的角色，作为一个Leader当然是要从企业/集团出发，但是作为个体又要从自身发展而考虑，所以并不矛盾

  • 领导力

    1. 视人为人是最大的领导力
    2. 共启愿景
    3. 挑战现状
    4. 使众人行
    5. 激励人心

    在这里本人理解到的是一支队伍共同奔向一个目标，和大学期间绑腿跑似乎很相似，一人不协调都有可能导致不能顺利到达终点，大学期间的班长一职让我也受益匪浅，一个班级就是一个小型企业，学会倾听，总结，行动，最终这个Team 才会有好的成绩。

  • 职业素养

    1. 自我管理
    2. 团队管理
    3. 业务管理

    对于Coder而言，可能自我管理是最重要的，当然还有业务管理，要对自己的业务掌握，并能够进行分析，拿出有效的解决方案，提高效率，同时也是提升自我的另一种方式，对于团队管理，只能结合实际想出一些实效方式，如果你想更深入管理，也可以针对性学习。

# 工作习惯

这里先强调下，每个人的生活方式不同，工作环境也不同，所以每个人工作方式也是不一样的，找到适合自己的才是最好的，无论是对自己，还是对Team。

  • 下沉管理，项目管理，重要岗位管理

  • 一个会议只解决一个问题

  • 不要做流水帐

  • 每日三个一

    1. 早会
    2. 晚会
    3. TDL(To Do List)

  • 每周三个一

    1. 夜校
    2. 周会
    3. 周报

  • 每月三个一

    1. 启动会
    2. 复盘会
    3. 团建活动

  • 每季度三个一

    1. 管理论台
    2. 绩效review
    3. 业务热点

  • review

    1. 确认谈话基调
    2. 创造良好环境，明确目标
    3. 跟踪进度

  • 裸心会

    1. 确认主题
    2. 团建活动
    3. 营造氛围
    4. 倾听观察
    5. 流程主线
    6. 层层裸心

    补充：裸心会开会的前提

    • 新团队
    • 团队有矛盾
    • 大家有压力
    • 协同有问题
    • 战略会
    • 季度常规复盘
    • 老板与团队断裂

以上工作习惯便是所学习到的，对于目前而言，能够达到的有每日，每周进行不同的工作总结，工作报告，以至于对Team能够很好的规划，还有就是每天自己都能review，view一天是否达到自己的期望，是否有学习，是否有突破，在这里我还是非常赞同对review的，在这个过程中，你会发现很多的不足，提升空间也会很大，对于我而言，时常有种罪恶感，源于自己好像有些虚度了，没有提升，哪怕只有那么一点点，在这里我说的是个人，对于Team就要对Project 规划，及目标任务的拆分，把控每个流程，其实一个人能力不仅仅在于学习能力，还有就是实践能力，对于Coder更好的是工程能力。

# 学习模型

  • 心智模型
  • 结构化思维
  • 刻意训练
  • 知识到结果之间差了一个太平洋
  • 管理靠练 听不会

如果你现在正管理着一个Team，那么不断的训练，无疑让你更好的掌握其中的技巧，但是如果你没有，或者作为Coder，可以尝试从自我管理开始，走出自己的第一步，循序渐进的实践，你也一定会受益颇丰。

最后我想分享给Coder应该具备的：

  • 编程能力

  • 架构能力

  • 工程能力
